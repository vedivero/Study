<협업의 세 가지 시나리오>

1. 나의 로컬 저장소 : 변경  / 원격 저장소 : 변경 사항 없음
	코드 작성하고 원격 저장소로 push
	$git push

	
2. 나의 로컬 저장소 : 변경 사항 없음 / 원격 저장소 : 변경

	로컬 저장소에 변경 사항을 pull로 동기화 후, push
	$git pull

3. 나의 로컬 저장소 : 변경 / 원격 저장소 : 변경

	방법.
	①rebase

	방법.②
	pull request (→merge)
	
	로컬 저장소에서 새로운 branch 생성하고 작업 후
	원격 저장소에 pull해주기를 요청 → pull request
	(ex. 야! 내가 방금 전에 새로운 commit을 만들었는데, 나의 새로운 버전을 받아줘! )

		로컬 저장소 기준 : push / 원격 저장소 기준 : pull
